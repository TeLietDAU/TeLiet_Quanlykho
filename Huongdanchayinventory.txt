## 📍 CHUẨN BỊ BAN ĐẦU

### 1️⃣ Bước 1: Bật MySQL Service

**Cách 1: Dùng XAMPP (Windows)**
- Mở XAMPP Control Panel
- Nhấn "Start" cho MySQL
- Đợi cho đến khi hiển thị "Running"

**Cách 2: Docker Compose (Tự động)**
```powershell
docker-compose up -d
docker-compose ps
```

**Cách 3: Windows Services**
```powershell
# Bật MySQL service trực tiếp
net start MySQL80
# hoặc MariaDB
net start MariaDB
```

### 2️⃣ Bước 2: Mở Terminal tại đúng thư mục

```powershell
# Mở PowerShell và cd vào project
cd D:\QLDH\TeLiet_Quanlykho
```

### 3️⃣ Bước 3: Đảm bảo dependencies đã cài

```powershell
# Cài pytest & pytest-django
pip install pytest pytest-django

# Hoặc cài từ requirements.txt
pip install -r requirements.txt
pip install pytest pytest-django
```

## ⚡ TỐI ƯU HÓA - CHẠY NHANH HƠN

### Keepdb Flag (Giữ lại Test Database)
Test lần đầu: ~40-75 giây (tạo DB)
Test lần 2+: ~5-10 giây (dùng lại DB)

```powershell
# Lần đầu tiên
python manage.py test apps.warehouse.tests --verbosity=2

# Lần thứ 2+ (QUICKER!)
python manage.py test apps.warehouse.tests --keepdb --verbosity=2
python manage.py test apps.order.tests --keepdb --verbosity=2
```

### Parallel Execution (Chạy song song với tất cả apps)
```powershell
python manage.py test apps.warehouse.tests apps.order.tests --keepdb
```

## 💾 DÙNG BATCH FILE (Windows)

Đã có 2 batch file sẵn để chạy tests nhanh:

```powershell
# Chạy tất cả tests
.\run_tests.bat all

# Chạy TEST với verbose mode
.\run_tests.bat verbose

# Chạy warehouse tests
.\run_tests.bat warehouse

# Lưu database để chạy nhanh hơn lần sau
.\run_tests.bat keep
```

## 🔍 XỬ LÝ LỖI THƯỜNG GẶP

### ❌ Lỗi 1: "Can't connect to server on 'localhost' (10061)"
**Nguyên nhân:** MySQL chưa bật
**Giải pháp:** 
```powershell
# Bật XAMPP hoặc chạy
docker-compose up -d
# hoặc
net start MySQL80
```

### ❌ Lỗi 2: "ModuleNotFoundError: No module named 'pytest'"
**Nguyên nhân:** pytest chưa cài
**Giải pháp:**
```powershell
pip install pytest pytest-django
```

### ❌ Lỗi 3: "Type 'yes' if you would like to try deleting the test database..."
**Nguyên nhân:** Test DB cũ vẫn tồn tại
**Giải pháp:** Gõ "yes" và Enter (safe - chỉ xóa DB ảo)

### ❌ Lỗi 4: AssertionError hoặc FAILED tests
**Nguyên nhân:** Code logic bị bug
**Giải pháp:**
1. Đọc error message kỹ
2. Kiểm tra models/services/repositories
3. Fix code rồi chạy lại

## 📈 WORKFLOW CHẠY TESTS CHUẨN

### Workflow 1: Development (Lần đầu)
```powershell
# 1. Bật MySQL
docker-compose up -d

# 2. Chạy tests lần đầu
cd D:\QLDH\TeLiet_Quanlykho
python manage.py test apps.warehouse.tests --verbosity=2

# 3. Nếu FAIL, fix code rồi chạy lại
# 4. Nếu PASS, chuyển sang tests khác
python manage.py test apps.order.tests --verbosity=2
```

### Workflow 2: Rapid Testing (đã tạo DB)
```powershell
# Chạy từ lần 2+ trở đi (NHANH HƠN)
python manage.py test apps.warehouse.tests --keepdb
python manage.py test apps.order.tests --keepdb

# Hoặc tất cả cùng lúc
python manage.py test apps.warehouse.tests apps.order.tests --keepdb
```

### Workflow 3: CI/CD (Đảm bảo sạch sẽ)
```powershell
# Chạy từ đầu, không dùng keepdb
python manage.py test apps.warehouse.tests apps.order.tests --verbosity=2
```
